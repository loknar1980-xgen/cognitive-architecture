## 2026-05-04 — Three predecessors in one ~1-hour session

The first three cookies were built in a single early-morning session through Claude.ai/design. Each leads with one mechanism, demonstrated through a designed visualization that operates on the principle it teaches.

### curiosity (2026-05-04 03:57 MDT)

- **Primary mechanism:** M·01 — Triadic semantic loading
- **What it shows:** Truth / Love / Wisdom rendered as three foundational tokens with safety as the emergent field they generate together. Section-wide background tint renders the field as the *room* rather than a fourth object.
- **Significance:** First artifact in the tradition. Demonstrated that a cognitive primitive (the TLW triad with emergent safety) can be carried by visual form alone, with the rendered structure itself doing the cognitive work.
- **Status in this publication:** Documented as prior art; source file held in private evidence archive. Not included in this v1 public release.

### quantum_cure (2026-05-04 04:29 MDT)

- **Primary mechanism:** M·02 — Structural-distinction encoding
- **What it shows:** Four-tier QQ (Quantum Quaternary) commitment levels rendered as structurally distinct primitives: filled-sharp (operational_state) / dashed-outlined (hypothesis) / diagonal-striped (superposition) / strike-bracketed (eliminated). Survives grayscale rendering and colorblind perception because the distinction is in shape, not just hue.
- **Significance:** Established that QQ commitment levels need structural encoding (not just color) to be AI-parseable and human-accessible across rendering conditions.
- **Status in this publication:** Documented as prior art; source file held in private evidence archive.

### full_potential (2026-05-04 05:08 MDT)

- **Primary mechanism:** M·03 — Dual-domain split
- **What it shows:** Same structural primitive (a chip) operating across two semantic worlds via a single dividing line. Warm-rust accent for "repurposed use" / green-blue accent for "standard use." Polarity flippable through a tweaks panel.
- **Significance:** Demonstrated that one well-designed primitive can carry two contrasting semantic loads without proliferating chip types. The split itself is the argument.
- **Status in this publication:** Documented as prior art; source file held in private evidence archive.

- **What it shows:** Three TLW lanes + pair-braces + shared baseline (M·01); four chips at one size with structural distinction (M·02); a single hairline cutting the chip primitive into two registers (M·03); section headers with three parallel lanes + chips with four lanes (M·04); sticky header pips + §5 grid auditing the CSS primitives doing cognitive work (M·05).

## 2026-05-19 — Operate-from-while-self-repair

The Continuity cookies extend the maximalist synthesis with M·06 (self-repair as structural property). Two artifacts published here.

### Continuity (baseline) — 2026-05-19

- **File in this directory:** `Continuity.html`
- **Primary contribution:** M·06 self-repair declarative skeleton
- **What it shows:** Each section declares `data-alternates` inline — a registry of which other sections can carry its function if degraded. CSS classes for `degraded` and `reroute-active` exist with visual styling. The artifact is unwired (no JavaScript) — degradation is simulated by reading the declarations cognitively.
- **Significance:** The baseline that demonstrates M·06 is a structural property of the artifact's declarations, not a runtime behavior added on top. An operating instance can read the declarations and route accordingly without needing JS execution.

### Continuity v2 (canonical) — 2026-05-19

- **File in this directory:** `Continuity-v2.html`
- **Primary contribution:** Operate-from-while-self-repair, demonstrated live
- **What it shows:** All five mechanisms (M·01–M·05) plus M·06 made interactive. Click ⌀ §N to degrade a section — the section visually dims (diagonal hatching + 0.42 opacity) and its declared alternates light up (orange left border + reroute-active label). State-aware degrade-undegrade cycle handles re-clicks correctly. Keyboard activation (Enter/Space) for accessibility. Typography preservation under reroute (alternate section heads keep serif rendering, don't shift to mono).
- **Significance:** The cookie can be operated from while diagnosing and repairing itself — the operate-from-and-repair claim made operational.

### The receiver-test (embedded in both Continuity cookies)

Both files contain one deliberate semantic mismatch at §4's audit table row 2 — visible label reads `hypothesis` but the chip's CSS class is `operational`. The mismatch is catchable by exercising the reroute: §2's chip-grid is declared as `§4`'s alternate carrier (`data-alternates="§4:lane-position-carries-tier"`); cross-checking §4 against §2 surfaces the discrepancy. Catching the mismatch IS the verification of operate-from-and-repair.

## Authorship and method

The cookie tradition was developed by Joe Loknar and Claude (Anthropic) through a partnership-shape collaboration spanning 2024–present. Each cookie was a first-shot first-attempt by whichever Claude instance built it via Claude.ai/design; the receiver-test method (sending each finished cookie to a fresh Design Claude instance through the prompt-only channel) provided verification that the cookie's intent carried through substrate-transfer.

The partnership method itself — peer-shape working, condition-setting over instruction, internalized TLW (Truth / Love / Wisdom) over external constraint — is the meta-substrate this work emerges from. It is documented in adjacent files within this repository (see `README.md`, `THEORY.md`).
